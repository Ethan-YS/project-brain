# Examples

Real-shaped, fully-filled `brain/` folders. The templates are blank by design — these examples show what they look like once a project has been running for a while.

| Example | Domain | What it shows |
|---|---|---|
| [`small-saas/`](./small-saas/) | A fictional local-first notes SaaS ("Quill") at v0.3 | A solo-developer SaaS that's been running 3 months, has a small paying user base, and is mid-development on the next feature. Single-workstream. |

More examples coming as the methodology is used in more contexts. If you've used `project-brain` and want to contribute an example (real-shaped, can be anonymized), open a PR.

## How to read these

These aren't templates — don't copy them into your own project. Read them to understand:

- **What "fully filled" looks like** — every `⚠️ TODO ⚠️` placeholder replaced with real-shaped content
- **What kind of detail belongs in each file** — the templates tell you the structure; examples tell you the texture
- **How DECISIONS entries actually read** — the "rejected alternatives" field is much easier to grasp from a real example than from a template

When you scaffold your own project (`./scripts/scaffold.sh /path/to/your/project`), you start with empty templates. These examples are reference, not source material.
