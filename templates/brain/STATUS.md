# STATUS — Current Development State

> **What this file answers**: "Where are we? Next step? What's blocked?"
>
> **Character**: **instantaneous state**. **Soft cap ~80 lines**. Safe to overwrite.
> Settle valuable content into `MAP.md` (structural changes) or `DECISIONS.md` (rationale) **first**, then overwrite.
>
> **vs `brain/topics/planning/`**:
> - planning = product roadmap (what features to build, big picture)
> - STATUS = development session state (what code we're writing right now, what we're stuck on)
>
> **vs `HANDOFF.md`**:
> - STATUS = steady state (overwrite at end of every session)
> - HANDOFF = transient bridge at the moment of window-switch (next one's generation archives the previous)
>
> **Who reads this**: every new session.
>
> **Multi-workstream (v2.1)**: copy this template as `STATUS_<workstream>.md` (e.g., `STATUS_dev.md`), one per workstream. See METHODOLOGY §3.5.
